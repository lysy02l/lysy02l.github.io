# 개인 프로젝트 개요

## 1. 프로젝트 개요

* **프로젝트명:** 
 보안 개념의 복습 및 정리를 위한 합법적 개인정보 탈취 실습
* **기간:** 
 2026 1학기 말까지(수정)
* **개발 환경:** (수정)

---

## 2. 프로젝트 목적

본 프로젝트는 머릿속에 단편적으로 존재하던 정보보안 관련 개념들을, 직접적인 실습을 통해 체계적으로 정리하는 것을 목표로 한다.

특히, 단순 이론 학습을 넘어서

* 취약한 시스템을 직접 설계하고
* 공격을 통해 취약점을 확인하며
* 이후 방어 기법을 적용하는 과정

을 통해 보안의 전반적인 흐름(공격 → 분석 → 방어)을 이해하고자 한다.

---

## 3. 진행 계획

| 주차   | 내용                                    |
| ------ | --------------------------------------- |
| 1주차  | 개발 환경 세팅 및 웹 서버 실행          |
| 2주차  | 기본 웹 페이지 및 로그인 UI 구성        |
| 3주차  | 데이터베이스 연동 및 사용자 정보 저장   |
| 4주차  | 로그인 기능 구현 및 세션 처리           |
| 5주차  | 의도적인 취약점 설계 (SQL Injection 등) |
| 6주차  | 취약점 공격 실습 및 분석                |
| 7주차  | 다양한 공격 기법 시도 및 기록           |
| 8주차  | 보안 강화 (입력값 검증, 해싱 등)        |
| 9주차  | 공격/방어 결과 비교 및 정리             |
| 10주차 | 최종 보고서 및 포트폴리오 작성          |

---

## 4. 실습 참여자 동의

본 프로젝트는 실제 환경이 아닌 테스트 환경에서 진행되며,
사용되는 모든 데이터는 가상의 정보 또는 사전에 동의를 받은 정보만을 사용한다.

또한, 실습 참여자는 프로젝트의 목적과 진행 방식에 대해 충분히 설명을 듣고 이에 동의하였음을 확인한다.

## 5. 예상되는 변수 요소

본 프로젝트 진행 중 다음과 같은 변수들이 발생할 수 있다:

* 개발 환경 설정 오류 및 기술적 문제
* 자기 객관화 부족으로 인한 실습 지연
* 예상치 못한 관계 파탄으로 인한 실습 참여자 상실

---

## 6. 기대 효과

* 웹 보안의 기본 구조 이해 (인증, 세션, 데이터 처리)
* 대표적인 취약점(SQL Injection 등)에 대한 실습 경험
* 공격과 방어를 모두 경험한 실질적인 보안 이해
* 향후 보안 관련 심화 학습을 위한 기반 마련
